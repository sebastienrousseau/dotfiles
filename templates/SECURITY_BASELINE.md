# Template Security Baseline

This document defines the security baseline for all project templates generated by `dot new`.

## Lockfile Policy

All templates include lockfiles by default to ensure reproducible builds and supply chain integrity.

| Template | Lockfile | Tool |
|----------|----------|------|
| `node` | `package-lock.json` | npm |
| `python` | (generate with `pip freeze > requirements.txt`) | pip |
| `go` | `go.sum` | go mod |

## CI Security Defaults

All templates include GitHub Actions workflows with security scanning:

| Template | Security Check | Tool |
|----------|---------------|------|
| `node` | Dependency audit | `npm audit --audit-level=high` |
| `python` | Vulnerability scan | `pip-audit --strict` |
| `go` | Vulnerability check | `govulncheck` |

## Template Security Guarantees

1. **No hardcoded secrets** - Templates use environment variable placeholders
2. **Lockfile-first** - Dependency versions are pinned
3. **CI-enforced security** - Vulnerability scans run on every PR
4. **Minimal dependencies** - Templates start with zero external dependencies
5. **Modern tooling** - Uses latest stable versions of build tools

## Verification

After running `dot new <lang> <name>`, verify:

```bash
# Node
cd <name> && npm audit

# Python
cd <name> && pip-audit

# Go
cd <name> && govulncheck ./...
```

## Extending Templates

When adding dependencies to generated projects:

1. Always update the lockfile
2. Run security scan before committing
3. Enable Dependabot for automated updates
4. Review dependency licenses for compliance
